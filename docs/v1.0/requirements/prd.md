# Textbook RAG Web - 产品需求文档 (PRD)

## 文档信息

- 版本: 1.0
- 作者: Product Manager
- 日期: 2026-03-07
- 状态: Draft

## 1. 产品概述

### 1.1 产品愿景

构建一个面向 AI/ML/NLP 教材学习场景的前后端分离 Web 应用，让用户在同一页面内完成教材阅读、自然语言提问、来源核验与原文定位，不再需要在多个页面或多个工具之间来回切换。

### 1.2 目标用户

- 学习 AI/ML/NLP 的学生
- 需要快速核对教材出处的助教或教师
- 需要验证答案可靠性的课程评审者

### 1.3 核心价值

- 可信回答: 答案基于教材检索结果生成，而不是无来源输出
- 深度溯源: 回答返回教材、章节、页码和位置索引
- 联动阅读: 左侧 PDF 浏览与右侧问答联动，点击来源即可跳转定位

## 2. 问题定义

### 2.1 用户问题

用户在阅读 AI/ML/NLP 教材时，经常面临三个问题：

- 不知道某个概念应该去哪本书、哪一章找
- 找到答案后难以快速验证原文出处
- 阅读 PDF 与问答工具分离，导致检索、核对、翻页成本高

### 2.2 当前方案缺口

- 现有项目方向已经具备教材、索引、RAG 和深度来源追踪基础，但缺少明确的 Web 产品形态
- 现有 Python 本地 UI 方案不利于形成清晰 API 契约和独立前端体验
- 来源追踪需要从“展示来源文本”升级为“点击后跳转到 PDF 对应位置”

### 2.3 为什么现在做

- 课程项目需要尽快形成可演示 MVP
- 深度来源追踪是项目差异化卖点，必须尽早固化到产品结构和接口中
- 当前仓库尚处于前期文档阶段，改成前后端分离的代价最低

## 3. 方案概览

### 3.1 方案描述

系统采用前后端分离 Web 架构：

- 前端提供双栏界面，左侧用于 PDF 阅读与定位，右侧用于问答交互
- 后端负责混合检索、RAG 编排、来源元数据返回和 PDF 定位数据输出
- 回答中的来源位置索引可点击，触发左侧 PDF 跳转到对应页码和区域

### 3.2 核心能力

- 自然语言提问与回答
- 混合检索与结果融合
- 回答级来源索引
- PDF 页码跳转
- bbox 高亮定位

### 3.3 用户主流程

1. 用户进入 Web 页面，左侧可浏览教材 PDF，右侧可输入问题。
2. 用户提交问题后，右侧显示回答、来源索引和候选引用片段。
3. 用户点击任一来源位置索引。
4. 左侧 PDF 跳转到对应页码。
5. 如果后端返回 bbox，PDF 同时高亮对应区域。

### 3.4 差异化

- 不是普通“聊天 + 引用书名”，而是“聊天 + 页级/区域级定位”
- 不是独立阅读器或独立问答器，而是联动双栏体验
- 不是单一检索方式，而是为教材问答优化的混合检索

## 4. 范围定义

### 4.1 In Scope

- 前后端分离 Web 架构
- 左 PDF / 右问答双栏布局
- 问答接口与来源索引接口
- 页码定位
- bbox 高亮能力的数据返回
- 基础筛选能力: 书名、章节、内容类型

### 4.2 Out of Scope

- 用户登录、多租户、权限体系
- 生产级云部署与运维体系
- 在线上传教材后台
- 自动化集成测试平台
- ROS 2 联动

### 4.3 MVP 定义

MVP 必须满足以下最小闭环：

- 能在浏览器中打开前端页面
- 能向后端发起教材问答请求
- 能返回答案与来源位置索引
- 能在左侧 PDF 中跳转到对应页码
- 若存在 bbox 数据，则完成区域高亮；若暂未实现 bbox 渲染，至少完成页码定位

## 5. Epic 列表

### EPIC-01 阅读与导航

目标: 用户可以稳定阅读 PDF，并在同一页面完成来源跳转。

### EPIC-02 问答与检索

目标: 用户可以提出教材相关问题并获得带来源的回答。

### EPIC-03 来源定位与核验

目标: 用户可以从回答中的来源位置索引直接定位到原文位置。

### EPIC-04 过滤与上下文联动

目标: 用户可以在问答、来源和 PDF 之间保持上下文连续。

## 6. 用户故事

### 6.1 P0 用户故事

#### US-001 提问并获得回答

作为学生，
我想要输入一个教材相关问题并得到回答，
以便快速理解概念而不必手动翻多本书。

验收标准:

- Given 用户在问答页面
- When 输入问题并提交
- Then 系统返回一条可阅读回答
- And 回答包含至少一个来源位置索引

#### US-002 查看来源元数据

作为学生，
我想要在回答中看到来源书籍、章节和页码，
以便判断回答是否可信。

验收标准:

- Given 用户已获得回答
- When 查看回答来源区域
- Then 可以看到书名、章节或节标题、页码
- And 可以看到来源片段摘要或引用文本

#### US-003 点击来源跳转 PDF

作为学生，
我想要点击回答中的来源位置索引后直接跳转到左侧 PDF 的对应位置，
以便快速核对原文。

验收标准:

- Given 页面左侧存在 PDF 浏览区且右侧已显示回答
- When 用户点击某个来源位置索引
- Then 左侧 PDF 跳转到对应页码
- And 当前定位状态在页面上有明确反馈

#### US-004 区域高亮

作为学生，
我想要在来源有坐标数据时看到对应区域被高亮，
以便快速确认命中内容就在页面的哪里。

验收标准:

- Given 后端返回来源 bbox 数据
- When 用户点击来源位置索引
- Then 左侧 PDF 在目标页渲染对应高亮区域
- And 高亮区域与来源索引一一对应

#### US-005 双栏联动浏览

作为学生，
我想要左边看 PDF、右边继续提问，
以便在阅读和问答之间保持连续上下文。

验收标准:

- Given 用户在桌面端访问应用
- When 页面加载完成
- Then 左右双栏同时可见
- And 用户无需跳转页面即可继续阅读或提问

### 6.2 P1 用户故事

#### US-006 按书名/章节过滤

作为学生，
我想要限制问答或来源范围到指定教材或章节，
以便缩小搜索范围。

验收标准:

- Given 页面提供过滤条件
- When 用户选择书名、章节或内容类型
- Then 后端查询范围按条件收窄
- And 回答与来源结果反映当前筛选条件

#### US-007 显示多个候选来源

作为学生，
我想要查看多个候选来源，
以便比较不同教材对同一问题的表述。

验收标准:

- Given 检索到多个相关来源
- When 回答生成完成
- Then 页面显示多个候选来源项
- And 每个来源项都可点击定位

#### US-008 错误与空结果反馈

作为学生，
我想要在查不到答案或服务失败时收到明确提示，
以便知道下一步怎么办。

验收标准:

- Given 查询为空或后端失败
- When 页面收到异常结果
- Then 显示明确错误或空结果提示
- And 提示用户可调整关键词、过滤条件或稍后重试

### 6.3 P2 用户故事

#### US-009 查询历史

作为学生，
我想要看到最近提过的问题，
以便快速回到之前的学习上下文。

#### US-010 前端上下文联动

作为学生，
我想要当前查看的 PDF、来源索引和问答上下文保持同步，
以便减少重复定位和重复提问。

## 8. 非功能需求

### 8.1 性能需求

- 首屏回答返回时间目标: P50 不超过 5 秒，P95 不超过 12 秒
- PDF 跳转反馈应在点击后 1 秒内可感知
- 页面交互不能因来源列表渲染而完全阻塞

### 8.2 可用性需求

- 双栏布局在桌面端默认同时可见
- 来源索引样式必须明显可点击
- 当前定位页码与选中来源需要有视觉反馈

### 8.3 兼容性需求

- MVP 优先支持桌面浏览器
- 现代 Chromium 内核浏览器需正常工作
- 移动端可在后续阶段降级为堆叠布局，不作为本阶段阻塞项

### 8.4 可靠性需求

- 后端异常时前端不可静默失败
- 缺少 bbox 时不能阻断页码跳转
- PDF 渲染失败时仍应保留来源信息可读

### 8.5 安全需求

- 不在前端暴露本地索引实现细节
- 前后端仅通过公开 API 通信
- 请求参数需进行基本校验

## 9. 验收标准

### 9.1 功能验收

- 所有 P0 用户故事完成
- 用户可至少完成一次“提问 -> 查看回答 -> 点击来源 -> 左侧 PDF 跳转”闭环
- bbox 存在时可完成高亮，否则至少完成页码跳转

### 9.2 非功能验收

- 关键错误路径有明确提示
- 前后端边界清晰，可进入架构设计
- PRD 中定义的接口字段可被后续架构阶段直接细化

## 10. 里程碑

### 10.1 MVP 范围

- P0 用户故事全部完成
- 本地全栈可运行
- 可演示来源跳转

### 10.2 V1.0 范围

- 完成 P0 + P1
- 支持基础过滤与多来源对比
- 提升联动体验与稳定性

### 10.3 未来规划

- 查询历史
- 用户系统
- 在线教材管理
- 外部系统集成

## 11. 风险与依赖

### 11.1 风险列表

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|----------|
| PDF 高亮实现复杂度高于预期 | 中 | 高 | 先保证页码跳转，bbox 高亮分层实现 |
| 检索结果质量不足导致来源不稳定 | 中 | 高 | 先做可演示问题集，后续调优检索融合 |
| 前端 PDF 渲染与跳转联动不稳定 | 中 | 中 | 选择成熟 PDF 渲染方案并优先验证最小闭环 |
| 本地模型或后端响应较慢 | 高 | 中 | 采用渐进加载与回答状态反馈 |

### 11.2 外部依赖

- 教材 PDF 数据与 MinerU 输出
- 混合检索索引
- 本地模型推理服务
- PDF 渲染组件/库

## 12. 术语表

- RAG: Retrieval-Augmented Generation
- bbox: PDF 页面中的矩形定位区域
- 来源位置索引: 回答中可点击的来源定位项，至少包含书名、页码和唯一定位标识
- 双栏界面: 左 PDF 阅读区 + 右问答交互区
